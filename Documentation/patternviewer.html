Pattern Viewer


The Pattern Viewer shows the pattern data of the playing module, just like it would look in a tracker. Each channel is shown as a column with the notes, instrument numbers, volume and effects. The row currently playing is highlighted and the view follows the playback in real-time. Not every player supports the Pattern Viewer yet, see Supported players for the details.

The display

At the top of the window, you find a header showing the module title together with the current position in the song, the pattern and row being played, the speed and BPM (the tempo of the music) and the number of tracks.

Below the header, you find the player control bar (if enabled) and the channel headers. Each channel header shows the channel number and a VU meter, which is a small bar that moves with the volume of the channel. Some module formats build their patterns out of separate tracks per channel. For these, the channel header also shows which track the channel is currently playing: e.g. #02 [003] means that channel 2 is playing track number 003. Formats that can transpose the notes additionally show the transpose value there, see the Transposed Notes description below.

Clicking on a channel header will mute or unmute that channel. If the module has more channels than fits in the window, small arrow buttons appear, which can be used to scroll through the channels.

Below the channel headers, the patterns are shown. While the module is playing, the view scrolls automatically, so the playing row always stays highlighted. When the player is paused, you can navigate through the patterns yourself by using the keyboard, see the list of keys below.

Context menu

Right-clicking anywhere in the pattern area opens the context menu, where the look and behaviour of the viewer can be changed. All settings are remembered between sessions.


Tracker Style

Selects the look of the viewer. The styles are grouped by category, and you can also switch between them with Ctrl+Shift and +/-.

The System styles Light and Dark show the patterns in a clean, modern look. Their Colored option draws notes, instruments, volume and effects in different colors, so the columns are easier to tell apart.

The other styles mimic classic trackers like ProTracker, AHX or OctaMED (Amiga) and FastTracker II, Scream Tracker or Impulse Tracker (PC) - more are added over time. They are adaptations, not exact copies: every style can show every module format, even with more channels, other effects or longer patterns than the original could handle. So you can even watch an XM file in the ProTracker look.
VU Meters

Controls the VU meters in the channel headers:

  • Off - disables the VU meters completely.
  • Note Kick - the meters jump every time a new note is triggered and then slowly fall back.
  • Real Volume - the meters show the actual volume of each channel instead, so fade-outs and other volume effects are also visible.

Below these modes, you can select the visual style of the VU meters. The styles mimic the look of the VU meters in the original trackers. If Match Tracker is selected, the VU meter style automatically follows the selected tracker style, so both always fit together.

Keep in mind that not every tracker had VU meters back then - the Pattern Viewer stays true to the original, so with Match Tracker enabled, such a tracker style is shown without VU meters as well.
Display Mode

Selects which columns to show for each channel. Some module formats can have more than one effect per channel and row:

  • Multi Effect Mode - shows all effects of each channel.
  • Single Effect Mode - shows only the first effect of each channel, which saves a lot of space.
  • Compact Mode - drops the effects and only shows the notes and instrument numbers.
  • Notes Only Mode - shows just the notes.

The narrower modes are useful for modules with many channels, so more channels fit in the window.
Scaling

Scales the whole display between 100% and 400%. This is useful on high resolution monitors or if you want to watch the patterns from a distance. You can also use Ctrl and +/- to change the scaling.

The Pattern Viewer draws with pixel fonts, like the original trackers did. At 100%, 200%, 300% and 400%, these are drawn natively - each pixel simply becomes 2, 3 or 4 pixels - which gives a pixel-perfect, sharp image. The sizes in between, like 125%, 150% and 175%, have to be scaled and can therefore look a little softer. That is technically unavoidable.
Player Control Bar

Shows or hides the player control bar at the top of the window.

The bar shows the playing time and position and has the same playback buttons as the main window: previous/next module, previous/next sub-song, skip backwards/forwards, restart, play/pause and stop. Clicking on the position bar jumps to that position in the module, so you can control the playback directly from the Pattern Viewer window.
Rolling Patterns

When enabled, the end of the previous pattern and the beginning of the next pattern roll into view, so the display scrolls seamlessly across pattern boundaries like one long song.

When disabled, only the current pattern is shown and the display jumps when a new pattern starts.
Hide Empty Normally, empty pattern entries are shown with placeholder characters. When this option is enabled, the placeholders are left out, so only the real data is shown. This gives a cleaner view and makes it easier to spot the notes in patterns that only hold a few of them.
Transposed Notes

Some module formats can transpose the notes while playing, e.g. by playing the same notes a few semi-tones higher at a later position in the song. For these modules, the current transpose value of each channel is shown in its channel header: e.g. #02 [+3] - or #02 [003+3] together with a track number - means that the notes of channel 2 are currently played 3 semi-tones up.

This option controls how the notes are shown:

  • Enabled - the notes are shown as they actually sound, with the transpose applied.
  • Disabled - the notes are shown as they are stored in the pattern data, which may differ from what you hear.

For modules without transpose, this option makes no difference.
Compress Patterns

Some modules use long patterns with a lot of empty space, e.g. when only every second or fourth row holds a note. When compression is On, the Pattern Viewer tries to show such patterns in a shorter form by leaving out the empty rows. Auto lets the player decide, which is the recommended setting.

Compression only works when the empty rows follow a regular scheme, e.g. every second row of the pattern is empty - patterns that do not fit such a scheme are always shown unchanged. For this reason it also cannot help with players where the whole song is converted into one single long pattern.
Row Numbers

Selects how the row numbers are shown:

  • Auto - the selected tracker style decides.
  • Decimal - normal decimal numbers.
  • Hexadecimal - base-16 numbers, the digits A-F follow after 9.
Instrument

Selects how the instrument numbers are shown:

  • Auto - the selected tracker style decides.
  • Decimal - normal decimal numbers.
  • Hexadecimal - base-16 numbers.
Volume

Selects how the volume values are shown:

  • Auto - the selected tracker style decides.
  • Decimal - normal decimal numbers.
  • Hexadecimal - base-16 numbers.
Track/Pattern

Selects how the track and pattern numbers are shown:

  • Auto - follows the row number format.
  • Decimal - normal decimal numbers.
  • Hexadecimal - base-16 numbers.
Grid Lines Shows or hides the grid lines between the channels (On / Off). Auto lets the selected tracker style decide.

Patterns and trackers

Most of the music NostalgicPlayer plays was made with so called trackers, like ProTracker on the Amiga or FastTracker II and Scream Tracker on the PC. In a tracker, the music is not stored as a recording. Instead it is built from small sound samples, which are played by notes arranged in patterns.

A pattern is a table: each column is a channel (also called a track) and each row is a moment in time. A cell can hold a note, the number of the instrument to play it with, a volume and one or more effects, e.g. to bend the pitch or fade the volume. The song itself is just a list that tells in which order the patterns are played.

The Pattern Viewer shows these patterns while the module plays, so you can see exactly how the music is put together, note by note. If you have never seen a tracker before, just watch the highlighted row while the music plays - you will quickly get the idea.

And if you have worked with one of the trackers back in the days, you will feel right at home, because the Pattern Viewer aims to show the data the same way the original tracker did. Keep in mind though, that the tracker styles are adaptations and not exact copies of the original programs, see the Tracker Style description above.

Supported players

The Pattern Viewer needs help from the player that plays the module, because every module format stores its data differently. Not every player supports this yet. Support is added player by player, so more and more formats will show their patterns in future versions. If the current module is played by a player without this support, an information page is shown instead of the patterns.

NostalgicPlayer always tries to present the music as patterns, no matter which format the module has. So whether you play a MOD, an XM or a completely different format, the patterns are always shown in the same familiar view. Even music formats that were not made with a tracker at all are converted to pattern form. For some of these formats, the whole song ends up as one single, long pattern - that is normal and simply how the conversion works for that format.
